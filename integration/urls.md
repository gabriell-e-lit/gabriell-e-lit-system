# urls.md

## Архитектура на URL в платформата gabriell‑e‑lit

### 1. Цел на документа

Този документ дефинира официалните правила за изграждане на URL в платформата gabriell‑e‑lit.
URL‑ите са стабилни, предвидими и произлизат от идентичността на обекта:

canonical name

display name

slug

ID

Документът гарантира, че всички URL‑и:

са последователни

са SEO‑оптимизирани

са устойчиви във времето

могат да бъдат генерирани автоматично

могат да бъдат използвани от локалния ИИ

### 2. Основни принципи

#### 2.1. URL = slug + структура
Всеки URL се изгражда от:

структура (папка / тип съдържание)

slug (латинизиран, уникален, стабилен)

Пример:

/authors/ivan-vazov

#### 2.2. URL е стабилен
След като бъде създаден, URL не се променя, освен при критична грешка.

#### 2.3. URL е латинизиран
Всички URL‑и използват латиница, независимо от езика на съдържанието.

#### 2.4. URL е уникален
Slug‑ът гарантира уникалност.
При колизия се добавя минимален суфикс:


/authors/ivan-vazov-2

#### 2.5. URL е SEO‑оптимизиран
URL‑ите са:

кратки

ясни

без стоп‑думи

без специални символи

### 3. Структура на URL по тип съдържание

#### 3.1. Автори

/authors/<slug>
Примери:


/authors/ivan-vazov
/authors/john-keats
/authors/charles-baudelaire

Правила:

slug = canonical name → латинизиран

няма поддиректории

няма ID в URL

#### 3.2. Публикации (книги, произведения)

/publications/<slug>
Примери:

/publications/pod-igoto
/publications/ode-to-a-nightingale
Правила:

slug = canonical title

преводните публикации използват slug на оригинала

няма езикови суфикси

#### 3.3. Галерия (картини, визуални активи)

/gallery/<slug>
Примери:

/gallery/sea-at-dawn
/gallery/self-portrait-1923
Правила:

slug = canonical name на произведението

изображенията не влияят на URL

#### 3.4. Колекции
Код
/collections/<slug>
Примери:


/collections/modernism
/collections/bulgarian-classics

#### 3.5. Изложби

/exhibitions/<slug>
#### 3.6. Списание gabriell‑e‑lit

Брой:

/magazine/issues/<slug>
Пример:


/magazine/issues/issue-34
Автори в броя:

/magazine/issues/<slug>/authors
Публикации в броя:

/magazine/issues/<slug>/publications
### 4. Връзка между canonical → slug → URL

#### 4.1. canonical name
Единствената стабилна идентичност.

#### 4.2. slug
Латинизирана, URL‑съвместима форма на canonical.

#### 4.3. URL
Структура + slug.

Пример:


canonical: "Иван Вазов"
slug: "ivan-vazov"
URL: /authors/ivan-vazov

### 5. Колизии

#### 5.1. Кога възникват
еднакви canonical имена

еднакви заглавия

преводни публикации със съвпадащи заглавия

#### 5.2. Решение
Добавя се минимален числов суфикс:

ivan-vazov-2
pod-igoto-2

#### 5.3. Никога не се добавят:
дати

езикови кодове

случайни символи

### 6. SEO правила за URL

####6.1. Забранени елементи
главни букви

интервали

специални символи

стоп‑думи

кирилица

#### 6.2. Разрешени елементи
малки букви

тирета

цифри (само при нужда)

###7. Примери

#### 7.1. Автор

canonical: "Чарлз Буковски"
slug: charles-bukowski
URL: /authors/charles-bukowski

#### 7.2. Публикация

canonical: "Под игото"
slug: pod-igoto
URL: /publications/pod-igoto

#### 7.3. Преводна публикация

canonical: "The Raven"
slug: the-raven
URL: /publications/the-raven

#### 8. Резюме

URL‑ите в платформата gabriell‑e‑lit са:

стабилни

латинизирани

SEO‑оптимизирани

предвидими

автоматично генерируеми

Те се изграждат от slug + структура и следват ясни, непротиворечиви правила, които гарантират устойчивост и автоматизация.

# AI_Library / system  
Модулната архитектура на системния слой за автоматизацията на gabriell-e-lit.com

Този пакет съдържа основните компоненти, които изграждат вътрешната логика на AI-базираната система за:
- извличане на данни от HTML
- нормализация на информация
- определяне на жанрове
- генериране на URL-и
- създаване на BookCell модели
- подготовка за статични страници

Архитектурата е напълно модулна и позволява лесно разширяване.

## 📦 Модули и тяхната роля

### 🔹 `adapters/book_cell/`
Съдържа адаптери, които преобразуват HTML структурата на сайта в вътрешни модели.

**BookCellCollectorAdapter_v1_0.py**  
- парсва `.book_cell` елементи  
- извлича сурови данни  
- нормализира формати  
- определя жанр (GenreResolver)  
- генерира slug (SlugifyService)  
- генерира URL за статична страница (BookPageURLResolver)  
- връща `BookCellModel_v2_0`

---

### 🔹 `models/book_cell/`
Описва структурата на данните.

**BookCellModel_v2_0.py**  
- основният модел за визуализация на книга

**BookCellModel_v2_0_builder.py**  
- конвертира суров речник → BookCellModel

---

### 🔹 `resolvers/`
Съдържа логика за определяне на ключови стойности.

**GenreResolver_v1_0.py**  
- определя жанра на книга по:
  - PDF/EPUB директория  
  - корица  
  - витрина  
  - fallback логика  

**BookPageURLResolver_v1_0.py**  
- генерира каноничния URL за статичната страница на книга

---

### 🔹 `services/`
Съдържа малки, независими услуги.

**SlugifyService_v1_0.py**  
- транслитерация български → латиница  
- нормализация на slug  
- премахване на тирета  
- специални правила за заглавия и автори  

---

## 🧩 Принципи на архитектурата

- **Модулност** – всеки компонент е самостоятелен  
- **Разширяемост** – лесно добавяне на нови версии (v1.1, v2.0…)  
- **Чистота** – адаптерите не съдържат бизнес логика  
- **Еднопосочен поток** – HTML → raw → model → renderer  
- **Стабилност** – централизирани услуги за slug, жанр и URL

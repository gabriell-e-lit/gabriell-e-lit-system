# DataTypes (v1 → v2.3)
## Нормативен слой за структурните модели на платформата gabriell‑e‑lit

Папката **datatypes/** съдържа всички версии на основните структурни модели, които описват
как изглеждат обектите в платформата:

- Author  
- Book  
- Artwork  
- Exhibition  
- Issue  
- Post  
- Section  
- EBookFile  

Това е **централната библиотека**, която определя формата на данните след екстракция,
нормализация и прилагане на приоритетите от слоя *dataflow/*.

---

# ## 1. Роля на DataTypes слоя

DataTypes определят:

- какви полета има всеки обект  
- какви типове данни се очакват  
- какви връзки съществуват между обектите  
- какви версии на моделите са използвани през времето  
- как екстракторът трябва да структурира финалния JSON  

Това е **единственият източник на истина** за структурата на данните.

---

# ## 2. Структура на папката

```
datatypes/
│
├── author/
├── book/
├── artwork/
├── exhibition/
├── issue/
├── post/
├── section/
└── ebookfile/
```

Всяка подпапка съдържа **всички версии** на съответния модел.

---

# ## 3. Версии на моделите

Моделите еволюират през времето.  
Всяка версия е запазена, за да може:

- екстракторът да бъде тестван срещу стари формати  
- да се проследява развитието на структурата  
- да се поддържат различни DataFlow версии  
- да се валидират исторически данни  

### Пример за еволюция:

**Author**
- v1.0 — базов модел  
- v2.1 — добавени портрети и галерии  
- v2.2 — добавени структурни страници, подкатегории, first_appearance  
- v2.3 — финализиран модел, съвместим с DataFlow v1.0

# ## 5. Връзка с DataFlow слоя

DataFlow документите определят:

- откъде идват данните  
- как се извличат  
- как се нормализират  
- как се прилагат приоритети  

DataTypes определят:

- **как изглежда крайният резултат**  

Тоест:

```
dataflow/  →  extractor  →  datatypes/
```

---

# ## 6. Как се използват DataTypes

- екстракторът валидира данните срещу актуалната версия  
- автоматизацията генерира JSON обекти по тези модели  
- статичният сайт и WordPress интеграцията използват тези структури  
- AI инструментите могат да генерират, валидират и сравняват обекти

# /integration/slugs.md

## Модул 2 от интеграционната архитектура

### Slug правила, латинизация и структурна консистентност в системата gabriell‑e‑lit

Този документ дефинира всички правила за slug генериране, латинизация, нормализация и структурна стабилност в системата gabriell‑e‑lit.
Slug е машинното име на обекта и е критичен за:

URL структура

SEO

вътрешна навигация

автоматизация

AI екстракция

консистентност между моделите

### 1. Основни понятия

#### 1.1. Slug (машинно име)
Slug е кратка, латинизирана, стабилна текстова форма, използвана в URL.

Свойства:
само малки латински букви

само тирета като разделители

без диакритики

без интервали

без пунктуация

без специални символи

стабилен във времето

Примери:
gabriela-zaneva

pismo-do-badeshteto

the-red-orchard

#### 1.2. Връзка между canonical name и slug
Slug се генерира единствено от canonical name.

Код
canonical_name → slug → url
Display name не участва в процеса.

### 2. Основни правила за slug генериране

#### 2.1. Латинизация
Всички slug стойности са на латиница, независимо от езика на оригинала.

Примери:
Габриела Цанева → gabriela-zaneva

Писмо до бъдещето → pismo-do-badeshteto

Червената овощна градина → chervenata-ovoshtna-gradina

#### 2.2. Малки букви
Всички букви се преобразуват в lowercase.

#### 2.3. Разделители
Интервалите се заменят с тире:

Код
"Gabriela Zaneva" → "gabriela-zaneva"

#### 2.4. Премахване на пунктуация
Премахват се:

запетаи

точки

кавички

удивителни

въпросителни

скоби

апострофи

#### 2.5. Премахване на специални символи
Премахват се:

%, &, $, #, @

*, +, =

/, \

: ;

#### 2.6. Кондензиране на тирета
Множество тирета се свеждат до едно:

Код
"pismo---do--badeshteto" → "pismo-do-badeshteto"

#### 2.7. Забрана за промяна
След като slug бъде създаден, той никога не се променя, освен при техническа грешка.

### 3. Специални правила за различни модели

#### 3.1. Автори
Slug = латинизиран canonical name.

Примери:

Габриела Занева → gabriela-zaneva

Haruki Murakami → haruki-murakami

#### 3.2. Публикации
Slug = латинизиран canonical name на произведението.

Примери:

Pismo do badeshteto → pismo-do-badeshteto

The Red Orchard → the-red-orchard

#### 3.3. Картини

Slug = латинизиран canonical name.

Пример:

Червената овощна градина → chervenata-ovoshtna-gradina

#### 3.4. Тагове
Slug = латинизиран canonical name на таг.

Пример:

италианска литература → italian-literature

### 4. Колизии и дисамбигуация

#### 4.1. Кога възниква колизия?
Когато два обекта имат еднакъв slug.

Пример:

spring (картина)

spring (публикация)

#### 4.2. Решение
Добавя се дисамбигуиращ елемент:

година

тип

инициали

номер

Примери:
spring-2021

spring-painting

ivan-petrov-writer

ivan-petrov-artist

### 5. Slug в JSON моделите

#### 5.1. Всеки модел съдържа slug
Пример:

Код
"slug": "gabriela-zaneva"
#### 5.2. Slug участва в URL
Пример:

Код
"url": "/authors/gabriela-zaneva/"

#### 5.3. Slug не участва във вътрешни връзки
Връзките използват ID, не slug.

### 6. Автоматизация

#### 6.1. AI екстракторът генерира slug от canonical name
Той:

латинизира

нормализира

премахва символи

кондензира тирета

валидира уникалност

#### 6.2. AI открива колизии
Ако slug вече съществува → предлага дисамбигуация.

#### 6.3. AI никога не променя slug автоматично
Промяна се прави само при:

техническа грешка

структурен конфликт

### 7. Примери

#### 7.1. Автор

#### 7.2. Публикация

}

#### 7.3. Таг
Код
{

  "slug": "italian-literature"
}

# Extractor‑PriorityRules – v1.0

## Глобални приоритетни правила за екстрактора

Този документ описва глобалните приоритети за избор на стойности при
екстракция на авторски данни от различните слоеве на платформата
gabriell‑e‑lit.

Правилата определят:

- кой източник е водещ за всяко поле  
- как се комбинират стойности  
- кога се използва fallback  
- кои източници имат абсолютен приоритет  

---

# 🟦 I. canonical_name  
**Приоритет на източниците:**

```
static_site > wp_structural > wp_tag > wp_post > ebooks
```

### ✔ Значение
- Статичният сайт е **канонът** за имена.  
- WordPress структурните страници могат да коригират, но не и да отменят канона.  
- wp_tag и wp_post са допълващи.  
- ebooks е най-слабият източник.

---

# 🟩 II. display_name  
**Приоритет на източниците:**

```
wp_post > wp_tag > H2 > H3 > static_site
```

### ✔ Значение
- Display name е **визуално име**, не канонично.  
- Най-често идва от публикации (wp_post).  
- H2/H3 са структурни, но не водещи.  
- Статичният сайт е fallback.

---

# 🟧 III. biography  
**Приоритет на източниците:**

```
H2 > H3 > wp_tag > wp_post > static_site
```

### ✔ Значение
- H2 е **абсолютен приоритет** (страници „Автори в брой X“).  
- H3 е втори по важност (структурни страници).  
- wp_tag и wp_post са допълващи.  
- static_site е архивен fallback.

---

# 🟪 IV. portrait  
**Приоритет на източниците:**

```
H2 > H3 > wp_tag > static_site > wp_post
```

### ✔ Значение
- Портретът следва почти същия модел като биографията.  
- static_site е по-надежден от wp_post за изображения.  

---

# 🟫 V. roles (сливане, не избор)

**Правило:**

```
roles = wp_structural (водещ)
       + static_site (канон)
       + wp_tag/wp_post (допълващи)
```

### ✔ Значение
- Ролите **не се избират**, а **се обединяват**.  
- wp_structural определя основната роля.  
- static_site добавя канонични роли (напр. художник, автор).  
- wp_tag/wp_post добавят контекстни роли (напр. преводач, редактор).

---

# 🟦 VI. gallery_subcategory  
**Приоритет на източниците:**

```
wp_structural > static_site
```

### ✔ Значение
- Подкатегориите на художници се определят от WordPress структурните страници.  
- Статичният сайт е вторичен източник.

---

# 🟩 VII. static_page_url  
**Единствен източник:**

```
static_site
```

### ✔ Значение
- Само статичният сайт съдържа реалните URL адреси на авторските страници.  
- WordPress не може да ги генерира.

---

# 🟧 VIII. source_urls  
**Правило:**

```
source_urls = обединение на всички източници
```

### ✔ Значение
- Това поле е **пълен списък** на всички места, откъдето е извлечена информация.  
- Използва се за проследимост, дебъг и автоматизация.

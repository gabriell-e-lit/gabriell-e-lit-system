# Priority Rules — Overview

## Част от слоя *ExtractionRules* на gabriell‑e‑lit

Папката **priorities/** съдържа всички нормативни правила, които определят
**приоритета на източниците** при екстракция на данни от различните слоеве на
платформата gabriell‑e‑lit.

Това е самостоятелен модул, който управлява:

- избор на стойности (canonical_name, display_name, biography, portrait)
- сливане на стойности (roles, source_urls)
- приоритет на структурните WordPress страници
- приоритет на статичния сайт
- fallback логика към wp_tag, wp_post и ebooks
- подкатегории на художници (gallery_subcategory)

# 📂 Структура на папката

```
priorities/
│
├── author/
│   ├── v1.1/
│   │   └── AuthorPriorityRules – v1.1.md
│   └── v1.2/
│       └── AuthorPriorityRules – v1.2.md
│
└── global/
    └── v1.0/
        └── Extractor‑PriorityRules – v1.0.md
```

### ✔ `author/`
Съдържа всички версии на правилата, които засягат **авторите**:

- H2/H3 логика  
- задължителни тагове  
- задължителни статични страници  
- приоритет на биография и портрет  
- подкатегории на художници  
- първа поява  

### ✔ `global/`
Съдържа глобални правила, които важат за всички типове обекти:

- canonical_name  
- display_name  
- roles (сливане)  
- static_page_url  
- source_urls  
- gallery_subcategory  

---

# 🧭 Версиониране

Всеки документ в тази папка има собствена версия:

- `v1.1`, `v1.2` за авторски правила  
- `v1.0` за глобалните правила  

Версиите се съхраняват в отделни подпапки, за да:

- се проследява еволюцията на правилата  
- се сравняват версии  
- се поддържа стабилност на екстрактора  
- се избягват breaking changes  

---

# 🧩 Роля в екстрактора

Priority Rules са **нормативен слой**, който:

- определя кое поле откъде се взема  
- решава конфликти между източници  
- управлява fallback логиката  
- гарантира консистентност между динамични и статични слоеве  
- служи като основа за автоматизацията  

Те НЕ съдържат:

- extraction logic  
- parsing правила  
- модели  
- JSON структури  

Те са **чисти правила**, които екстракторът следва.

---

# ⭐ Финал

Папката **priorities/** е критичен модул от слоя *ExtractionRules*.  
Тя съдържа всички приоритетни правила, разделени по тип (author/global) и версия.

Това е нормативният източник на истина за всички приоритети в екстрактора.

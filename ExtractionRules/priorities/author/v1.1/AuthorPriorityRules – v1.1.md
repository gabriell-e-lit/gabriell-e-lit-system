# AuthorPriorityRules – v1.1
## Приоритетни правила за автори (структурни WordPress страници)

Този документ описва приоритетите и задължителните изисквания за автори,
идентифицирани в структурните WordPress страници на платформата gabriell‑e‑lit.

---

# 🟦 1. H2 автори → задължителни биографии в WordPress таговете

H2 автори се срещат само в страниците **„Автори в брой X“**.

```
if author appears as H2 in "Автори в брой X":
    must_have_wp_tag = true
    // ако има биография в тази страница → тя става основна biography
    // wp_tag трябва да съдържа биографията (копиране/синхронизация)
    // first_appearance = тази страница или съответния Issue
```

### ✔ Значение
- Това е **най-високият структурен маркер** за автори в WordPress.
- Биографията от H2 страницата е **основна**.
- Тагът на автора е **задължителен**.

---

# 🟩 2. H3 автори → задължителни статични страници

H3 автори се срещат в структурни страници като:

- Автори на издателството  
- Автори в е‑библиотека  
- Автори в е‑галерия  
- Гост‑редактори  

```
if author appears as H3 in structural pages:
    must_have_static_page = true
    // статичната страница трябва да бъде създадена/свързана
    // static_author_url не трябва да бъде празно
```

### ✔ Значение
- H3 не е по-нисък приоритет — просто е структурно ниво.
- Авторът **задължително** трябва да има статична страница.

---

# 🟧 3. Причина за H3 (структурна, не приоритетна)

H3 се използва, защото страниците имат H2 нива като:

- година  
- роля  
- брой  
- колекция  

Това **не намалява важността на автора**, а само описва структурата.

---

# 🟪 4. Приоритет на източниците за biography и portrait

```
priority_for_biography_and_portrait:
    1. H2 страници "Автори в брой X"
    2. H3 структурни страници (издателство, е-библиотека, е-галерия, гост-редактори)
    3. wp_tag
    4. wp_posts
    5. static_site
    6. ebooks
```

### ✔ Значение
- H2 винаги има **абсолютен приоритет**.
- H3 е втори по важност.
- Всички останали източници са fallback.

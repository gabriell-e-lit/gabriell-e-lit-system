SYSTEM‑ARCHITECTURE‑v1.0
Обединена архитектура на платформата gabriell‑e‑lit
Версия: 1.0 (официална)

I. МЕТА‑АРХИТЕКТУРА
1. Основен принцип
gabriell‑e‑lit е многослойна екосистема, в която:
- WordPress = динамика
- Статичният сайт = канон
- Поддомейните = структурирани архиви
- e‑books = пълни текстове и метаданни
- Екстракторът = обединяващ слой
- DataTypes v2 = единен модел на истината

Всеки слой е източник на истина за различен тип данни.
Нито един слой не е вторичен.

II. СЛОЕВЕ НА СИСТЕМАТА
1. ROOT LAYER (gabriell-e-lit.com)
Основни статични страници, описателно ядро, без динамични зависимости.

2. PROJECT LAYER
Три входни точки:
- Издателство (WordPress)
- Списание (WordPress)
- Поддомейни (e-library, e-gallery, e-books)

3. DYNAMIC LAYER 1 — Издателство (WordPress)
- WP Pages
- WP Categories
- Publisher Genres
- AuthorIndex
- Връзки към поддомейните

4. DYNAMIC LAYER 2 — Списание (WordPress)
- Главна категория
- Теми
- Рубрики
- Направления
- Архиви
- Гост‑редактори
- Автори в брой

5. STATIC LAYER — Статичен сайт
- Статични автори
- Статични книги
- Статични изложби
- Статични галерии
- Статични архиви

6. SUBDOMAIN LAYER — e-library / e-gallery / e-books
Най‑структурираният слой: реални файлове, изображения, корици, изложби.

V. ПРИОРИТЕТИ
1. Биография
H2 > H3 > wp_tag > wp_posts > static_site > ebooks

2. Портрет
H2 > H3 > wp_tag > static_site > wp_posts

3. Име
static_site > wp_structural > wp_tag > wp_post

4. Роли
wp_structural (водещ) + static_site + wp_tag/wp_post

5. Галерийна подкатегория
wp_structural > static_site

VI. КАНОН
1. Канонът включва:
- статични авторски страници
- статични книги
- статични изложби
- галерии
- редакторски утвърдени биографии
- редакторски утвърдени портрети

2. Канонът НЕ включва:
- WordPress публикации
- WordPress тагове
- автоматично генерирани данни

3. Канонът е редакторска власт.
Екстракторът може да предлага, но не може да променя.

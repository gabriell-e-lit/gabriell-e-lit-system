ARCHITECTURE OVERVIEW — v2.3
gabriell-e-lit-library / system / architecture

Общ архитектурен преглед на системата gabriell-e-lit.
Документът описва слоевете, ролите, зависимостите и принципите на платформата.

------------------------------------------------------------
I. PURPOSE OF THE ARCHITECTURE
------------------------------------------------------------
Архитектурата на gabriell-e-lit е изградена върху два основни слоя:

1) Нормативен слой (Normative Layer)
   - Човешки, концептуален, редакторски.
   - Описва как трябва да бъде устроена системата.
   - Съдържа правила, структури, принципи, карти, модели (концептуални).

2) Технически слой (Technical Layer)
   - Машинен, формален, структурен.
   - Описва как автоматизацията работи.
   - Съдържа модели, правила за екстракция, правила за генерация.

Тези два слоя са отделени, но взаимосвързани.

------------------------------------------------------------
II. LAYER STRUCTURE (6-слойна архитектура)
------------------------------------------------------------

1) NORMATIVE LAYER (редакторски)
   - Architecture Rules
   - DataFlow Architecture
   - Navigation Rules
   - Slug Rules
   - Priority Rules
   - Migration Rules
   - Conceptual Models

2) TECHNICAL LAYER (машинен)
   - AuthorModel v2.3
   - BookModel v2.3
   - PublicationModel v2.3
   - NavigationModel v2.3
   - Extractor Rules
   - Generator Rules

3) TEMPLATE LAYER (визуален)
   - AuthorTemplate v2.3
   - BookTemplate v2.3
   - PublicationTemplate v2.3
   - Include blocks (author_block, book_block, etc.)

4) CONTENT LAYER (данни в страниците)
   - Горният панел на страниците
   - Данни за автори, книги, публикации, навигация

5) AUTOMATION LAYER
   - Extractor v2.3
   - Generator v2.3

6) DIRECTORY LAYER (файлова структура)
   - /includes/
   - /p-izdatelstvo/includes/
   - /p-izdatelstvo/authors/
   - /p-izdatelstvo/books/
   - /p-izdatelstvo/publications/
   - /AI-Library/
   - /gabriell-e-lit-library/

------------------------------------------------------------
III. FLOW OF CONTROL (как тече логиката)
------------------------------------------------------------

NORMATIVE LAYER
    ↓ определя
TECHNICAL LAYER (models + rules)
    ↓ определя
TEMPLATE LAYER (визуални блокове)
    ↓ определя
CONTENT LAYER (data panels)
    ↓ се чете от
AUTOMATION LAYER (extractor + generator)
    ↓ създава
CONTENT LAYER (нови страници)
    ↓ се съхранява в
DIRECTORY LAYER (файлова структура)

------------------------------------------------------------
IV. KEY PRINCIPLES
------------------------------------------------------------

1) Старите страници не се пипат.
2) Новите страници живеят в p-izdatelstvo/.
3) Данните са в горния панел.
4) Визуализацията е в include-и.
5) Екстракторът чете, но не променя.
6) Генераторът създава, но не презаписва.
7) Нормативният слой е човешки.
8) Техническият слой е машинен.
9) Двата слоя са отделени, но съгласувани.
10) Всичко е модулно, разширяемо и устойчиво.

------------------------------------------------------------
V. ROLE OF THE TWO LIBRARIES
------------------------------------------------------------

gabriell-e-lit-library (нормативна)
    - .txt файлове
    - архитектура, правила, концепции
    - за редактора

AI-Library (техническа)
    - .md файлове
    - модели, машинни правила, JSON структури
    - за автоматизацията

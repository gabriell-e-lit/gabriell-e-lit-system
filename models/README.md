# /integration/README.md
## Интеграционен слой на системата gabriell‑e‑lit

### 1. Роля на интеграционния слой
Папката `/integration` съдържа архитектурните документи, които описват:

- как работят моделите заедно  
- как се генерират идентификатори  
- как се генерират slug стойности  
- как се изграждат URL  
- как се управлява идентичността  
- как AI екстракторът автоматизира процесите  

Това е „мозъкът“ на системата — мястото, където се дефинират правилата, които управляват цялата платформа.

---

### 2. Модули в интеграционния слой

#### 2.1. identifiers.md — Идентичност и идентификатори
Съдържа:
- правила за canonical name  
- правила за display name  
- правила за ID  
- диапазонна архитектура  
- ръчен контрол върху идентичността  
- правила за преводни произведения  
- правила за кирилизация на автори в превод  

#### 2.2. slugs.md — Slug архитектура
Съдържа:
- правила за латинизация  
- slug → URL  
- стабилност на slug  
- колизии и дисамбигуация  
- slug при преводни произведения  
- slug при ръчно зададен canonical name  

(Следващи модули ще бъдат добавени тук.)

---

### 3. Как работи интеграционният слой

#### 3.1. Canonical → Slug → URL
Всички URL се генерират автоматично от canonical name.

#### 3.2. ID → Връзки
Всички връзки между модели използват ID, никога slug или имена.

#### 3.3. Display name → Визуално име
Display name е контекстно зависим:
- оригинал → латиница  
- превод → кирилица  

#### 3.4. Ръчен контрол
Авторите могат да имат ръчно зададени:
- canonical name  
- slug  

Това гарантира идентичност.

---

### 4. Роля в автоматизацията
Интеграционният слой определя:
- как AI екстракторът разпознава автори  
- как се генерират slug  
- как се валидират ID  
- как се откриват колизии  
- как се създават JSON файлове автоматично

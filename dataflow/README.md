# DataFlow — Overview

## Поток на данните в платформата gabriell‑e‑lit

Папката **dataflow/** съдържа всички документи, които описват движението на
данните между слоевете на платформата gabriell‑e‑lit:

- WordPress (динамичен слой)
- Екстрактор (структурен слой)
- JSON модели (междинен слой)
- Локален ИИ (нормализация и корекции)
- Детска тема (генератори)
- Статични страници (краен слой)

Това е **оперативната архитектура** на системата — как данните текат, как се
трансформират и как се генерират финалните страници.

---

# 🟦 1. Основна концепция

DataFlow описва **как данните се движат**, а не как са структурирани.

Това включва:

- входни точки  
- трансформации  
- нормализации  
- приоритети  
- изходни точки  

DataFlow е „кръвоносната система“ на платформата.

---

# 🟩 2. Основни етапи на потока

## 2.1. WordPress → Екстрактор
Екстракторът:

- извлича автори, книги, броеве, изложби  
- прилага приоритетни правила  
- нормализира имена, роли, биографии  
- създава структурирани JSON обекти  

## 2.2. JSON → Локален ИИ
Локалният ИИ:

- валидира  
- коригира  
- попълва липсващи данни  
- генерира структурирани блокове  

## 2.3. Локален ИИ → Детска тема
Детската тема получава:

- Author.json  
- Book.json  
- Exhibition.json  
- Artwork.json  
- Issue.json  

и ги превръща в:

- страници  
- списъци  
- навигация  

## 2.4. Детска тема → Статични страници
Генерира:

- авторски страници  
- страници за книги  
- изложби  
- галерии  
- азбучни списъци  

---

# 🟧 3. Двупосочни връзки

DataFlow описва и обратните връзки:

- WordPress ← статични страници (синхронизация)  
- WordPress ← локален ИИ (попълване на липсващи данни)  
- WordPress ← екстрактор (актуализация на тагове)  

---

# 🟪 4. Роля на DataFlow в автоматизацията

DataFlow е **централният документ**, който:

- свързва всички слоеве  
- определя реда на операциите  
- гарантира консистентност  
- служи като основа за екстрактора  
- служи като основа за генераторите  

Без DataFlow няма автоматизация.

---

# ⭐ Финал

Папката **dataflow/** е оперативният слой на архитектурата.  
Тя описва как данните се движат, трансформират и генерират в платформата
gabriell‑e‑lit.

Това е основата на автоматизацията.

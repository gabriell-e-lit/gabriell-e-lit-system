## 1. Обхват на документа
Този документ описва как екстракторът обработва **WordPress Tags**, за да създаде и обогати обектите от DataTypes v2:

- Author  
- Book  
- Artwork (в редки случаи)  

Документът покрива **само WordPress тагове**, не категории, не страници, не постове.

---

## 2. Роля на WordPress таговете в системата

WordPress таговете имат **една основна роля**:

# ✔ Да дефинират автори, които имат публикации в списанието.

Това е единственият източник, който:

- гарантира, че авторът има WordPress присъствие  
- съдържа биография  
- съдържа портрет  
- съдържа линкове към публикации  

Таговете са **задължителни** за всички автори, които са H2 в „Автори в брой X“.

---

## 3. Какво представлява един WordPress таг за автор

Типичният таг съдържа:

- име на автора  
- кратка биография  
- портрет (featured image или изображение в съдържанието)  
- списък с публикации  
- линкове към статични страници (ако има)  

Екстракторът третира всеки таг като:

```
AuthorTag {
    name,
    biography,
    portrait,
    posts[],
    source_url
}
```

---

## 4. Поток: WordPress Tag → Author

### 4.1 Разпознаване
Екстракторът разпознава авторски тагове по:

- URL структура (`/tag/име-на-автор/`)  
- наличие на биография  
- наличие на публикации  

### 4.2 Данни, които се извличат
- **name** → заглавието на тага  
- **biography** → текстът в описанието на тага  
- **portrait** → featured image или първо изображение  
- **posts[]** → всички публикации, свързани с тага  
- **source_url** → URL на тага  

### 4.3 Ефекти върху Author
```
must_have_wp_tag = true
biography_priority = wp_tag_if_no_H2
portrait_priority = wp_tag_if_no_H2
posts += всички постове от тага
source_urls += url_of_tag
```

## 5. Връзка с AuthorPriorityRules v1

### 5.1 Приоритет на биографията
```
H2 > H3 > wp_tag > wp_posts > static_site > ebooks
```

### 5.2 Приоритет на портрета
```
H2 > H3 > wp_tag > wp_posts > static_site
```

### 5.3 Приоритет на името
```
H2/H3 > wp_tag > wp_posts
```

## 6. Специални случаи

### 6.1 Автор има таг, но не е H2/H3 в структурни страници
Тогава:

```
Author.must_have_wp_tag = true
Author.must_have_static_page = false
```

Това е автор, който е публикувал в списанието, но не е част от издателството, е‑библиотеката или е‑галерията.

---

### 6.2 Автор е H3 в структурна страница, но няма таг
Тогава:

```
Author.must_have_static_page = true
Author.must_have_wp_tag = false
```

Това е автор, който е част от издателството или е‑библиотеката, но не е публикувал в списанието.

---

### 6.3 Автор е H2 в „Автори в брой X“, но няма таг
Това е **грешка**, която екстракторът трябва да маркира:

```
ERROR: Missing required WordPress tag for H2 author.
```

---

## 7. Финален резултат
След обработка на всички WordPress тагове, екстракторът има:

- биографии за автори, които не са H2/H3  
- портрети за автори, които не са H2/H3  
- списък с публикации за всеки автор  
- гаранция, че всички H2 автори имат таг  
- допълнителни източници за Author v2.3  

---

## 8. Несъответствия и пропуски (важно)

### 8.1 Автор има таг + публикации, но липсва в „Автори в брой X“
Това е редакторски пропуск, не грешка.

Екстракторът трябва да:
```
Author.is_missing_in_issue_pages = true
Author.must_have_wp_tag = true
Author.must_have_static_page = false
```

И да добави в диагностичния отчет:
```
WARNING: Author has tag and posts but is missing from issue author pages.
```

Авторът НЕ се изключва и НЕ се маркира като грешен.

---

### 8.2 Автор е H2 в „Автори в брой X“, но няма таг
Това е липсващ елемент, не грешка.

Екстракторът трябва да:
```
Author.is_missing_wp_tag = true
Author.must_have_wp_tag = true
```

И да добави в диагностичния отчет:
```
WARNING: H2 author is missing required WordPress tag.
```

Авторът НЕ се изключва и НЕ се маркира като невалиден.

---

### 8.3 Автор е H3 в структурни страници, но няма таг
Това е нормално.

Екстракторът трябва да:
```
Author.must_have_static_page = true
Author.must_have_wp_tag = false
```

---

### 8.4 Автор има таг, но няма публикации
Това е допустимо (например автор с биография, но без публикуван текст).

Екстракторът трябва да:
```
Author.has_posts = false
Author.must_have_wp_tag = true
```

---

### 8.5 НИКОГА не се трият автори или тагове
Екстракторът:
- не премахва автори  
- не премахва тагове  
- не маркира автори като „невалидни“  
- не изключва автори от системата  

Всички несъответствия се докладват, но не водят до загуба на данни.
